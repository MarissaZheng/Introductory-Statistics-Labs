Reading Datasets

Today we’ll be looking at a dataset containing information about passengers on the H.M.S. Titanic.

Loading the data

The first step of any data analysis in R will be to “load” the data. This means taking a dataset from online or from a folder on your computer and putting that information into the R program.

This dataset is already stored online, at my personal website The information is stored in “csv” format, or “comma-separated values”. We will simply use an R function called read.csv() to get this information and store it.

When you load a dataset, you will need to give it a name. This is so that in the future, you can tell R where to find the information it needs - especially once you are dealing with more than one dataset.

Throughout this class, we will do our best to follow the Tidyverse Style Guide, which gives some guidelines for making your code look nice and easy to read.

The style guide suggests giving names that use only lowercase letters and underscores (“_“). The name should also be descriptive; for example, for a dataset containing incomes of workers in the United States, we might go with us_incomes_2018 rather than incomes or my_dataset.

Cleaning Datasets

It is very rare that data comes to us in exactly the format we hope, or that R interprets the variable exactly as it should. You will nearly always need to clean the data a bit before you begin calculating or plotting anything. We will practice with one of the common issues today.

Missing Data

Sometimes, values in the dataset are not present; perhaps because they were misrecorded or unavailable.

You, as the scientist, will need to decide how to handle these missing values. This will depend on your research question and your choice of analysis. For now, we will simply toss out the rows (passengers) for which we are missing some information.

The function na.omit() will automatically drop all rows from a dataset that have any values of NA in them.

Oh no! They are still there! What happened?

We have fallen into a common mistake when using the %>% approach: we did not actually save our updated dataset. That is, we told R, “Please do na.omit() to titanic”, but we did not use <- to tell it “… and save the results under this name”.

Although we could come up with another name - perhaps titanic2 - and create a second dataset to work with, that isn’t really necessary. Most of the time, we will simply save our updated data with the same name as the old one.

While convenient, this updating can be dangerous. We have now replaced the dataset titanic with a new version. If we made a mistake, our original data is gone!

If this happens at home, you can always simply reload your data by running read.csv() again. For the purposes of this lab, a copy of the original dataset called titanic_orig is already loaded into R. Any time you want to “reset” your data, simply run the line

titanic <- titanic_orig

in one of your code boxes.

Quantitative and Categorical variables

Notice that when you ran summary(titanic), you were given the mean and 5-number summary of the variable called Pclass. This is because the information for that variable is given as number corresponding to the cabin class of the passenger (1, 2, or 3). When R encounters numbers, it treats the variable as quantitative. We know, however, that this should be categorical and ordinal.

To fix this, we will mutate() our dataset, which means adding or changing a variable. The function factor() will turn a numeric variable into categories.

Visualizing Categorical Variables

Data loaded? Check! Data cleaned? Check! Time for the fun stuff. Let’s begin by making some nice plots for the categorical variables in our dataset.

Bar Plots

We will be using a package called ggplot2 to make all of our visualizations. This package has a very specific format for plotting.

First, we tell R that we want to make a plot from a certain dataset, and we specify the aethetic, aes(), which determines the variables we want to visualize.

Then, we add the type of plot we want, and any extra options.

Beyond the crucial information of the dataset and the variable(s) to plot, there are also plenty of optional settings to make our image nicer.

Now suppose we wish to compare two categorical variables. One option is to simply make separate barplots for each category. We do so by adding the option facet_grid() to our plot. Try it below.

Another option is to make a stacked bar chart, which combines the two categorical variables on the bars themselves. Note that these charts continue to obey the area principle - the area of the colored sections corresponds to the number of people in that category!
